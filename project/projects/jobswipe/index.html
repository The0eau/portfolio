JobSwipe AI – AI-Powered Career Assistant

Role: Lead Developer · Team: 4 · Stack: Python, React, Gradio, OpenAI API · Focus: AI Product, LLMs, UX, Matching

Overview

JobSwipe AI streamlines job applications by generating tailored CVs, cover letters, and interview questions from a user profile and a job description. A matching ranker scores roles and suggests application strategy (what to apply to, how, and when).

  • Projected to reduce time-to-offer by ~30% through automation and guidance.
  • Integrated LLM interview simulator for practice with adaptive feedback.
  • Built an interactive Gradio UI for instant upload → preview → download flows.

Problem & Users

Candidates waste hours tailoring materials, struggle to prioritize roles, and get little feedback. JobSwipe AI targets:

  • Students & early-career: fast, guided applications with high-quality outputs.
  • Career switchers: skill translation and gap-centric cover letters.
  • Busy professionals: batch processing and interview rehearsal on demand.

Key Features

  • Smart Matching Ranker — Scores job–candidate fit with weighted signal fusion (skills, seniority, responsibilities, keywords).
  • Auto-Tailored CV & Cover Letter — Content distilled from the user’s profile, achievements, and the JD with role-specific emphasis.
  • Interview Simulator — LLM-generated, role-specific questions; real-time feedback on clarity, structure (STAR), and depth.
  • One-click Gradio Workflow — Upload CV + JD → get ranked roles, AI outputs, and interview practice in minutes.
  • Versioning — Save, compare, and iterate on multiple CV/CL versions per role.

How It Works

  1. Ingest — User uploads CV (PDF/Doc) and pastes a job description.
  2. Parse — Extraction & normalization (skills, metrics, entities, dates).
  3. Match — Ranker computes fit score and identifies gaps/opportunities.
  4. Generate — LLM drafts CV variants, cover letter, and interview questions.
  5. Review — User edits in-app and exports ready-to-apply assets.

Architecture (High-Level)

  • Frontend: React SPA embedding a Gradio app for rapid prototyping of flows and previews.
  • Orchestration: Python services coordinating parsing, ranking, and LLM prompts.
  • LLM Layer: OpenAI API with prompt templates and guardrails for consistency.
  • Matching: Hybrid scoring (keyword overlap, semantic embeddings, seniority heuristics).
  • Storage: User session artifacts (drafts, feedback, exports) persisted server-side.

Results & Impact

  • −30% time-to-offer (projected) via automation and better prioritization.
  • Faster first draft: CV/CL drafts in minutes vs hours; higher output quality.
  • Better prep: Interview simulator drives structured answers and confidence.

Note: Some figures are based on pilot tests and modeled scenarios; full live-user study planned.
